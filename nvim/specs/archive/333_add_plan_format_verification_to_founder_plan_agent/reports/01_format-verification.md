## Summary

The core `planner-agent.md` already has a Stage 6a verification step (lines 282-299) that checks plan artifacts for required metadata fields before writing success metadata. The `founder-plan-agent.md` lacks this verification, meaning plans generated by the founder planning agent could be missing required fields without detection.

## Findings

### Reference Pattern: planner-agent.md Stage 6a

The core planner-agent verifies 4 metadata fields at Stage 6a:
- `- **Status**: [NOT STARTED]`
- `- **Task**: {N} - {title}`
- `- **Effort**:`
- `- **Type**:`

If any field is missing, the agent edits the plan to add it before proceeding.

### Gap Analysis: founder-plan-agent.md

The founder-plan-agent currently has:
- Stage 0: Initialize Early Metadata
- Stage 1: Parse Delegation Context
- Stage 2: Locate and Read Research Report
- Stage 3: Parse Research Report
- Stage 4: Determine Report Type
- Stage 5: Generate Plan Artifact
- Stage 6: Write Plan File
- Stage 7: Write Metadata File
- Stage 8: Return Brief Text Summary

There is no verification step between Stage 6 (Write Plan File) and Stage 7 (Write Metadata File).

### plan-format.md Required Fields and Sections

**Required metadata fields** (8 total):
1. Task
2. Status
3. Effort
4. Dependencies
5. Research Inputs
6. Artifacts
7. Standards
8. Type

**Required sections** (7 total):
1. Overview
2. Goals & Non-Goals
3. Risks & Mitigations
4. Implementation Phases
5. Testing & Validation
6. Artifacts & Outputs
7. Rollback/Contingency

**Phase format requirements**:
- Heading: `### Phase N: {name} [STATUS]`
- Per-phase fields: Goal, Tasks (checklist), Timing

### Recommended Enhancement

Add a new Stage 6a between the current Stage 6 (Write Plan File) and Stage 7 (Write Metadata File) that:
1. Re-reads the written plan file
2. Verifies all 8 required metadata fields exist
3. Verifies all 7 required sections exist
4. Verifies phase heading format under `## Implementation Phases`
5. Verifies per-phase fields (Goal, Tasks, Timing)
6. Edits the plan to add any missing fields/sections before proceeding

The verification should be more comprehensive than planner-agent's (which only checks 4 fields) since we now have a complete specification in plan-format.md.

## Recommendation

Proceed to planning and implementation. The change is well-scoped: add a single Stage 6a section to founder-plan-agent.md and update the Critical Requirements section.
