# Research Report: Remaining Root Files in .claude/

**Task**: 467 - Move remaining root files to extensions/core/
**Date**: 2026-04-16
**Session**: sess_1776400640_d16198
**Type**: meta

## Summary

After task 465 moved ~203 core agent system files into `.claude/extensions/core/`, 10 files remain in `.claude/` root directories that are core extension content rather than loader infrastructure. These files would be incorrectly synced as root-level content when loading core into a new repository.

## Principle

The `.claude/` root directories should contain only loader infrastructure -- files needed by the extension system before any extension loads. All agent system content should live in `extensions/core/` and be copied to the appropriate locations when core is loaded via the picker.

## Code Changes Required

### 1. Move files with `git mv`

8 files total across 3 categories:
- 5 context documentation/schema files
- 2 systemd files
- 1 root README.md

### 2. Update init.lua

The `core-index-entries.json` path is hardcoded in init.lua (~line 496):
```lua
local core_index_path = target_dir .. "/context/core-index-entries.json"
```
This needs to read from the extension source directory instead, or be handled via the existing `merge_targets` mechanism.

### 3. Add `copy_systemd()` to loader

The loader currently has `copy_scripts`, `copy_hooks`, `copy_docs`, `copy_templates`. It needs a `copy_systemd()` function (or reuse a generic copy function) to handle systemd files.

Alternatively, `systemd` could be added as a provides category in the manifest and handled by an existing generic copy path.

### 4. Handle README.md at root

Two options:
- **Option A**: Add README.md as a `merge_targets` entry (like CLAUDE.md) and generate/copy it on load
- **Option B**: Add it to the `root_file_names` sync list and copy from `extensions/core/docs/README.md` on load

### 5. Update sync.lua

The `root_file_names` list in sync.lua includes `README.md`. After moving README.md to `extensions/core/docs/`, sync needs to source it from the extension directory.

### 6. Update manifest.json

Add `systemd` to the core manifest's `provides` section. Verify `docs` provides includes README.md.

## Relationship to Task 466

Task 466 (convert `core-index-entries.json` to standard `merge_targets`) is a follow-up that changes HOW the file is consumed. This task (467) moves WHERE the file lives. Both are needed, but 467 should execute first since it's about physical file location, while 466 is about the loading mechanism.

## Post-Move Verification

And `.claude/systemd/` should be empty (populated on core load).
