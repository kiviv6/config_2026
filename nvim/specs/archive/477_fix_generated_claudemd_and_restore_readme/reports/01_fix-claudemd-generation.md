## Executive Summary

- The duplicate `# Agent System` header in `.claude/CLAUDE.md` (lines 1 and 10) is caused by the header template and the core merge-source both starting with the same heading.
- The `.claude/README.md` was moved to `.claude/extensions/core/docs/README.md` by task 467, but two references to `@.claude/README.md` in the generated CLAUDE.md were not updated.
- A copy also exists at `.claude/docs/README.md` -- either location could serve as the canonical path for a symlink or restore.
- The root `README.md` (project-level Neovim documentation) is not referenced anywhere in the generated CLAUDE.md and could benefit from a reference in the project overview section.
- The `generate_claudemd` function in `merge.lua` is well-structured but the header/fragment separation was not coordinated with the core fragment content.

## Context & Scope

The `.claude/CLAUDE.md` file is a computed artifact generated by the neotex extension loader (`<leader>ac`). It is assembled by the `generate_claudemd()` function in `lua/neotex/plugins/ai/shared/extensions/merge.lua` (line 548). The function concatenates a header template with content fragments from each loaded extension.

This research investigates:
1. The duplicate header bug
2. The missing `.claude/README.md` file
3. Broken `@.claude/README.md` references in generated content
4. Potential improvements to the generation process

## Findings

### 1. Duplicate Header Root Cause

**Location**: `lua/neotex/plugins/ai/shared/extensions/merge.lua`, function `generate_claudemd()` (line 548)

**Mechanism**: The function builds the output by:
1. Reading the header template from `extensions/core/templates/claudemd-header.md`
2. Reading the core merge-source from `extensions/core/merge-sources/claudemd.md`
3. Concatenating: header + core fragment + extension fragments

**The bug**: Both files start with `# Agent System`:
- `templates/claudemd-header.md` line 1: `# Agent System`
- `merge-sources/claudemd.md` line 1: `# Agent System`

Result in the generated file (lines 1-10):
```
# Agent System

This file is generated automatically from loaded extensions. Do not edit directly.

<!-- Generated by: neotex extension loader -->
<!-- Edit source files in .claude/extensions/*/merge-sources/claudemd.md or EXTENSION.md -->


# Agent System
```

**Fix options** (choose one):
- **Option A** (recommended): Remove `# Agent System` from `merge-sources/claudemd.md` since the header template already provides it. The header template is the canonical location for the top-level heading and generation metadata.
- **Option B**: Remove `# Agent System` from the header template and keep it in the merge source. Less clean since the header template exists specifically to provide the file preamble.

### 2. Missing .claude/README.md

**History**: Task 467 (`70db03dd`) moved `.claude/README.md` to `.claude/extensions/core/docs/README.md`. The file was NOT deleted -- it was relocated.

**Current state**: The same content exists at TWO locations:
- `.claude/extensions/core/docs/README.md` (moved here by task 467)
- `.claude/docs/README.md` (appears to be the synced/installed copy)

Neither is at `.claude/README.md`.

**Impact**: The generated CLAUDE.md contains two `@.claude/README.md` references that point to a non-existent file:
- Line 3 of core merge-source: `For comprehensive documentation, see @.claude/README.md.`
- Line 10 of core merge-source: `- **Architecture**: @.claude/README.md`

**Fix options**:
- **Option A** (recommended): Restore `.claude/README.md` as a symlink to `docs/README.md`, or simply restore the file at the original path. The `.claude/docs/README.md` is the installed copy. Creating `.claude/README.md` (as a copy or symlink) fixes the broken references without changing the generation code.
- **Option B**: Update the two references in `extensions/core/merge-sources/claudemd.md` to point to `@.claude/docs/README.md`. Simpler but changes the reference convention.
- **Option C**: Add `.claude/README.md` to the core extension's `provides.root_files` so it gets installed during extension loading. Most robust long-term.

### 3. Root README.md Reference

The root `README.md` at `/home/benjamin/.config/nvim/README.md` documents the Neovim configuration project (installation, features, file structure). The generated CLAUDE.md does not reference it.

The root `CLAUDE.md` (at project root) does reference `README.md` indirectly through the project structure, but the generated `.claude/CLAUDE.md` has no link to it.

**Recommendation**: Add a reference to the root README in the "Project Structure" section or "Context Imports" section of the core merge-source. Something like:
```markdown
## Context Imports

Core context (always available):
- @README.md
- @.claude/context/repo/project-overview.md
```

### 4. Generation Process Review

**Code quality**: The `generate_claudemd()` function (lines 548-660 of merge.lua) is clean and well-documented. Key observations:

- **No duplication detection**: The function does not check whether fragments contain duplicate headings relative to the header template. This is by design (fragments are trusted), but the current bug shows the risk.
- **Two code paths**: The extension loader uses `generate_claudemd()`, but the sync operation (`sync.lua` line 243) still uses `inject_section()` which has section markers and a different merge strategy. This dual approach could cause inconsistency if sync is run without a subsequent `generate_claudemd()` call.
- **Seed behavior**: `inject_section()` (lines 108-119) seeds from a sibling `README.md` if the target doesn't exist. Since `.claude/README.md` is now missing, this seed would produce an empty file if `inject_section` were called when CLAUDE.md doesn't exist. This is a latent bug in the sync path.
- **No root README reference**: As noted above.

**Additional improvements**:
- The header template could include a timestamp or version indicator (e.g., `<!-- Generated: {ISO8601} -->`) for debugging generation issues.
- The "Quick Reference" section duplicates the `@.claude/README.md` reference; after fixing the broken reference, consider consolidating.

## Decisions

- The duplicate header is definitively caused by content overlap between the header template and the core merge-source fragment.
- The README.md was intentionally moved by task 467, not accidentally deleted. The broken references are collateral damage from that move.

## Recommendations

1. **Remove duplicate heading** from `extensions/core/merge-sources/claudemd.md` (delete line 1 `# Agent System` and the blank line after it). Priority: high.

2. **Restore `.claude/README.md`**: Either restore the file at `.claude/README.md` or update the two `@.claude/README.md` references to `@.claude/docs/README.md`. Priority: high.

3. **Add root README reference**: Add `@README.md` to the Context Imports section of the core merge-source. Priority: medium.

4. **Fix sync path seed**: Update `inject_section()` seed logic to account for the moved README. Priority: low (generate_claudemd is the primary path now).

5. **Consider adding generation timestamp**: Add `<!-- Generated: {date} -->` to the header template. Priority: low.

## Risks & Mitigations

- **Risk**: Removing the heading from `claudemd.md` changes the fragment's standalone readability.
  **Mitigation**: The fragment is not meant to be read standalone; it is a merge source. The header template provides the heading.

- **Risk**: Restoring `.claude/README.md` may conflict with task 467's intent to consolidate files into `extensions/core/`.
  **Mitigation**: The simplest fix is to update references rather than restoring the file. Either approach works.
