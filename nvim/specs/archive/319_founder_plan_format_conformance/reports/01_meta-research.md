## Root Cause Analysis

### Issue 1: Missing plan-format.md Context Reference

`founder-plan-agent.md` (lines 29-42) loads these contexts:
```markdown
**Always Load**:
- `@.claude/extensions/founder/context/project/founder/domain/business-frameworks.md`
- `@.claude/extensions/founder/context/project/founder/patterns/mode-selection.md`
- `@.claude/extensions/founder/context/project/founder/patterns/legal-planning.md`
- `@.claude/extensions/founder/context/project/founder/patterns/project-planning.md`

**Load for Output**:
- `@.claude/context/formats/return-metadata-file.md`
```

**Missing**: `@.claude/context/formats/plan-format.md`

### Issue 2: Custom Non-Conforming Plan Structure

The agent's Stage 5 (lines 251-345) defines a completely custom plan structure. Key differences:

1. **Header format**: Uses separate lines for metadata instead of bulleted list
2. **Section names**: Uses custom section names (Phases, Success Criteria, Report Output)
3. **Phase format**: Uses Objectives/Inputs/Outputs instead of Goal/Tasks/Timing

### Issue 3: index-entries.json Missing Core Format

The founder extension's `index-entries.json` only includes founder-specific context. The core `plan-format.md` is not included in the `load_when.agents` for `founder-plan-agent`.

## Recommended Fix

### Step 1: Update founder-plan-agent.md Context References

Add to "Always Load" section:
```markdown
- `@.claude/context/formats/plan-format.md` - Standard plan artifact format
```

### Step 2: Update index-entries.json

Add entry or update existing core index.json to include:
```json
{
  "path": "formats/plan-format.md",
  "load_when": {
    "agents": ["founder-plan-agent", "planner-agent", ...],
    "commands": ["/plan"]
  }
}
```

### Step 3: Rewrite Stage 5 in founder-plan-agent.md

Change the plan generation template to conform to standard:

```markdown
# Implementation Plan: {title}
- **Task**: {id} - {title}
- **Status**: [NOT STARTED]
- **Effort**: {estimate}
- **Dependencies**: {list or None}
- **Research Inputs**: {report path}
- **Artifacts**: plans/MM_{short-slug}.md
- **Standards**:
  - .claude/context/formats/plan-format.md
  - .claude/context/standards/status-markers.md
- **Type**: {report_type}

## Overview
{summary from research, 2-4 sentences}

## Goals & Non-Goals
- **Goals**: {from research findings}
- **Non-Goals**: {scope boundaries}

## Risks & Mitigations
- Risk: {identified risk} Mitigation: {approach}

## Implementation Phases
### Phase 1: {name} [NOT STARTED]
- **Goal:** {single statement}
- **Tasks:**
  - [ ] {task 1}
  - [ ] {task 2}
- **Timing:** {duration}

## Testing & Validation
- [ ] {criterion 1}
- [ ] {criterion 2}

## Artifacts & Outputs
- plans/MM_{short-slug}.md
- summaries/NN_{short-slug}-summary.md

## Rollback/Contingency
- {recovery plan}
```

### Step 4: Adapt Founder-Specific Content

Keep founder domain content (Research Integration, Report Type sections) but structure within standard format:
- Research Integration goes in Overview section
- Report Type becomes the `Type` metadata field
- Mode goes in metadata or Overview
