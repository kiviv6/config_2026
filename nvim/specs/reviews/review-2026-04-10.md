## High Priority Issues

### Extension manifest routing blocks missing
**Files**: 11 extension manifests
**Description**: `.claude/scripts/check-extension-docs.sh` reports FAIL on 11 extensions because their `manifest.json` declares skills but has no `routing` block mapping task types to those skills.

Affected extensions:
- filetypes (5 skills)
- formal (4 skills)
- latex (2 skills)
- lean (4 skills)
- memory (1 skill)
- nix (2 skills)
- nvim (2 skills)
- python (2 skills)
- typst (2 skills)
- web (3 skills)
- z3 (2 skills)

**Impact**: When these extensions are loaded via `<leader>ac`, the research/plan/implement routing table cannot dispatch tasks to their skills. Commands fall back to the default skills, which means extension domain knowledge is not applied even when the extension is active.

**Recommended fix**: Add `routing.research`, `routing.plan`, `routing.implement` blocks to each manifest, mapping each declared task_type to its corresponding skill. Pattern established by `present/` and `founder/` extensions.
